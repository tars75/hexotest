JavaScriptについて学ぶ - 14

投稿日 2018-04-24 | In JavaScript , DOM

JavaScriptについてのメモ第14回目。今回はイベントについて。

イベントリスナー / Event Listeners

UIとWebページのインタラクションを実装するには、イベントリスナーを使う。

DOM上のエレメントなどにイベントリスナーをつけることで、そこで特定のイベントが発生したかどうかを拾って、イベントが発生したときの動作を決めることができる。

DOM上のすべてのエレメントに対してイベントリスナーをつけることができる。


書き方は下記の通り。

.addEventListener(パラメータ1, パラメータ2);

  • パラメータ1には何のイベントをトリガーにするかを指定。

  • パラメータ2にはコールバックする関数を指定。

  • イベントが発生した際は、まず関数がコールバックされ、そのあとにデフォルトのビヘイビアがあればそれが行われる。（例えばそのエレメントがリンクであれば、関数が実行された後にデフォルトのビヘイビアとしてリンク先のページに飛ぶ。）

  • 関数のパラメータにイベントオブジェクトを引数として指定でき、それに対して.preventDefault()を使うことでデフォルトのビヘイビアを防ぐことができる。


イベントオブジェクト / The Event Object

イベントオブジェクトは、DOM上で発生したイベントを表す。

何かしらイベントが発生すると、イベントオブジェクトが生成される。

イベントオブジェクトにはいろいろな情報が含まれており、例えばイベントが発生したエレメントや座標、イベントが発生したときにシフトキーが押されていたかどうかなど、書きれないほどたくさんの情報をもつ。
