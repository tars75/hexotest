JavaScriptについて学ぶ - 8

投稿日 2018-04-11

JavaScriptについてのメモ第8回目。今回はいよいよWindowオブジェクトについて。

Window Object

Windowオブジェクトはブラウザのウインドウ本体を表し、ブラウザとJSをつなぐ一番の大本となるオブジェクト。

ブラウザやウェブページはすべて、オブジェクトのツリー構造によって表現されている。
「ウインドウのなかの、htmlエレメントのなかの、headエレメントの中の、pエレメントの〜、」といった形でブラウザやWebページの要素をオブジェクトのつらなりとして扱えるようになっていて、その一番上にあるのがこのWindowオブジェクト。

ウェブページのツリー構造の標準化されたモデル自体をDOMとよび、documentオブジェクトによってそのモデル構造が実現されている。
その中にはすべてのhtmlエレメントに対応したプロパティ、すべてのhtmlエレメントにアクセスできるメソッド、すべてのhtmlイベントなどが含まれている。
このdocumentオブジェクトもまた、Windowオブジェクトに含まれている。

とりあえずコンソールでwindowと入力してみると、ずらっとWindowオブジェクトに含まれているものの一覧がでてくる。

  • alert()
  • documentオブジェクト
  • fetch API
  • innerHeight
  • innerWidth

などなど。とにかく数え切れないくらいでてくる。

Windowオブジェクトのすべてをメモするのは不可能なので、今回はさらっと触りだけメモする。

alertメソッド

windowオブジェクトのなかでabc順的に一番最初にくるメソッド。警告ダイアログを出す。
なんとなく90年代のイメージがあるメソッド。

promptメソッド

ユーザー入力ダイアログをだして、その入力を受け取れるメソッド。
これもwindow.は省略してかける。

confirmメソッド

okボタンとキャンセルボタンのあるダイアログをだして、okが押されたらtrueを返す。

Outer Height と Outer Width

ウインドウの一番外側の高さや幅を取得するプロパティ。読み取り専用。

Inner Height と　Inner Width

ウインドウの内側、ウェブページそのものが表示されているところの高さや幅を取得するプロパティ。読み取り専用。

Scrollポイント

縦方向または横方向にどれだけスクロールされているかを取得するプロパティ。読み取り専用。

Location オブジェクト

クライアントのホストやドメイン、ポートなど、ネット上での場所に関する情報を取得するオブジェクト。

hrefプロパティにアドレスを入れるとRedirectされる。

Reloadメソッドでページの再読み込みができる。

History オブジェクト

今のウインドウで読み込んだページの履歴を取得するオブジェクト。

ブラウザの戻るボタン的なメソッドがある。

Navigator オブジェクト

ブラウザの種類やバージョン、言語など、JSが動いているブラウザの情報を取得するオブジェクト。

ただ、MDNをみるとブラウザの検出という行為自体できる限り避けるように提唱されているようなので要注意。

ブラウザ同士の競争でuserAgentがごちゃごちゃになってしまったことなどが背景にある様子。
