DOMツリーの探索 - DOM Traversal

DOMのノードには、親ノードを取得したり、子ノードを取得したり、兄弟ノードを取得したりするプロパティが用意されている。

使いこなせば一つのノードを起点に、DOMツリー上を縦横無尽に行ったり来たりできる。

ここではDOMのノードで使える、DOMツリーを行ったり来たりするためのプロパティをメモ。


子ノード

    • 子ノードのノードリスト。エレメント以外のノードも含んだ全ての子ノードのリスト。

    • htmlのファイルの中で、タグ間に空白や改行、コメントなどがあればそれもtextノードとして含まれるので注意。

    • ほとんどの場合、textノードは不要なので下記の.childerenプロパティの方が使いやすい。

  • .children

    • 子ノードのHTMLコレクション。

    • エレメントのみで、エレメント以外のノードは含まれないので、エレメントだけ欲しいときに有効。

  • .firstChild

    • 一番最初の子ノードを取得。エレメント以外も含まれる。

    • たいていの場合、エレメントではなくtextノードなので、一番最初の子エレメントを取得した場合は下の.firstElementChildを使う。

  • .firstElementChild

    • 一番最初の子エレメントを取得。

  • .lastChild

    • 一番最後の子ノードを取得。エレメント以外も含む。

  • .lastElementChild

    • 一番最後の子エレメントを取得。

    • 子エレメントの数を数える。エレメント以外のノードの数は含まないので、空白や改行のテキストノードなどを気にせず普通に使える。

  • .parentNode

    • 親ノードを取得する。たいていの場合は親ノードはエレメント。

  • .parentElement

    • 親エレメントを取得する。たいていは親ノードはエレメントなので、.parentNodeでもこれでも同じ結果に成る場合が多い。

    • エレメント以外も含むので注意。

    • 次の兄弟エレメントを取得する。エレメント以外は含まない。

    • 前の兄弟ノードを取得する。エレメント意外も含むので注意。

    • 前の兄弟エレメントを取得する。

DOM セレクター ( Single Elements用 )

Documentオブジェクトには、DOMからHTMLエレメントを取得するためのDOMセレクターが用意されている。

ここではDOMセレクターのうち、シングルエレメント用（エレメントを単体で取得するもの）についてメモ。


  • document.querySelector()

    • CSSのセレクタでエレメントを取得できるDOMセレクター。

    • なのでIDでもクラスでもネストされたエレメントでも、もっと言えば nth-child みたいなやつでも指定できる。

    • あくまでもシングルエレメント用なので、複数該当するエレメントがあっても、取得されるのは最初のエレメント1つだけなので注意。

    • はっきりいってこれがあればgetElementById()はいらない気がするけど、よく出てくるのでいちおう両方ともメモする。

  • document.getElementById()

    • メモするまでもなく文字通りそのまま、IDからエレメントを取得するDOMセレクター。


getElementByIdでもquerySelectorでも、DOMセレクターでエレメントを取得した後はそのエレメントにいろいろ変更をかけたりすることができ、例えばCSSのスタイルや中身のテキストなども変更できる。


  • getElementById()でも、querySelector()でも、取得したオブジェクトを変数に渡して格納できる。

  • .styleと続けてからCSSのプロパティを指定することで、CSSのスタイルを変えることができる。

DOM セレクター ( Multiple Elements用 )

今度はDOMセレクターのうち、複数のエレメントを取得する用のものについてメモ。


  • document.getElementsByClassName()

    • 文字通り、クラス名からエレメントを取得する。

    • HTMLコレクションで取得されるため、配列のメソッドなどを使いたい場合は配列に変換する必要がある。

  • document.getElementsByTagName()

    • こっちは文字通りタグ名でエレメントを取得する。

    • これもHTMLコレクションで取得されるため、配列のメソッドなどを使いたい場合は配列に変換する必要がある。

  • document.querySelectorAll()

    • document.querySelector()が複数のエレメント取得に対応したもの。

    • CSSセレクタで当てはまるもの全てを取得できる。

    • ノードリストとして取得されるため、上記2つとは違って変換しなくてもそのまま配列のメソッドが使える。

（window.は省略できる。）

Window Object

Windowオブジェクトはブラウザのウインドウ本体を表し、ブラウザとJSをつなぐ一番の大本となるオブジェクト。

ブラウザやウェブページはすべて、オブジェクトのツリー構造によって表現されている。
「ウインドウのなかの、htmlエレメントのなかの、headエレメントの中の、pエレメントの〜、」といった形でブラウザやWebページの要素をオブジェクトのつらなりとして扱えるようになっていて、その一番上にあるのがこのWindowオブジェクト。

ウェブページのツリー構造の標準化されたモデル自体をDOMとよび、documentオブジェクトによってそのモデル構造が実現されている。
その中にはすべてのhtmlエレメントに対応したプロパティ、すべてのhtmlエレメントにアクセスできるメソッド、すべてのhtmlイベントなどが含まれている。
このdocumentオブジェクトもまた、Windowオブジェクトに含まれている。

  • alert()

関数宣言 - Function Declarations

関数とは、なんらかの機能をもったコードのかたまりのこと。基本的に関数を宣言するだけではそのコードは実行されない。関数を使いたいところで呼びだすことでそのコードを実行できる。

パラメーターを設定し、関数を呼びだすときに値（引数と呼ばれる）を渡して、その値を関数の中で使うことができる。また関数の中で処理した結果を出力して渡すこともできる。（返り値やreturnなどと呼ばれる。）

パラメーターは,で区切ることで複数設定でき、またパラメーターなしでも関数を作れる。

パラメーターのある関数を呼ぶときに、引数を何も渡さないと、パラメータはundefinedになる。


関数のデフォルトパラメータ - Default Function Parameters

上記で記載したように、パラメータになにも引数を渡さないと、undefinedになる。
これを防ぐために、パラメーターに対して引数を渡さなかった場合のデフォルト値を設定することができる。ES6からは従来と比較してかなり簡単に書ける。


関数式 - Function Expressions

Function Expressionsとは、基本的に関数を他の変数に代入するときに使われる構文を指す。
書き方は関数宣言とほとんど同じだが、関数宣言との違いとして、関数名を省略できるという特徴がある。また関数宣言とは違って最後にセミコロン;が必要なので注意。


IIFEs - Immediately Invokable Function Expressions

直訳すると、即座に呼び出し可能な関数表現。
IIFEsとは文字通り、関数宣言時に呼び出しもおこなってそのまま実行できる関数のこと。もっとざっくり言うと、宣言と実行を同時にできる関数の書き方。

ちなみにIIFEは英単語のiffyと同じような発音（カタカナで書くとたぶんイフィーが近い）で呼ばれ、IIFEsはそれにsをつけてイフィースまたはイフィーズなどと発音される。

名前はなんだか長くて難しそうだけど、書き方自体はかんたんで、()の中に無名な関数の宣言をかいて、そのあとにまたさらに呼びだすための()をつけるだけ。この呼びだすための()を書かないと実行されないのでそこは要注意。

Truthy or Falsy

JSではすべての変数がTruthyかFalsyに分けられる。booleanに変換したときにTrueになる値がTruthy、Falseになる値がFalsyと呼ばれる。

Falsyな値：

  • false
  • 0 and -0
  • '' and "" (空っぽの文字列)
  • null and undefined
  • NaN
  • document.all

上記以外はすべてTruthyな値。

比較 - Comparison

二つの値や型を比較する条件式の構文まとめ。比較演算子と呼ばれる。

  • ==
    二つの値が等しいかチェックする。データ型が異なる場合、自動的に変換した上で比較される。等しければtrueを返す。

  • !=
    二つの値が等しくないかをチェックする。データ型が異なる場合、自動的に変換した上で比較される。等しくなければtrueを返す。

  • ===
    データ型と値の両方を比較し、どちらも等しければtrueを返す。

  • !==
    データ型と値の両方を比較し、等しくないところがあったらtrueを返す。

  • >
    左が右より大きければtrueを返す。

  • >=
    左が右より大きいか等しければtrueを返す。

  • <と<=
    それぞれ>と>=の逆。

論理演算子 - Logical Operators

条件式2つを比較して値を返す構文。

  • 論理AND
    条件1 && 条件2
    条件1がFalsyな場合は条件1を返す。
    条件1がTruthyな場合は条件2を返す。
    よって、条件がどちらともTruthyのときのみTruthyな値がリターンされる。

  • 論理OR
    条件1 || 条件2
    条件1がTruthyな場合は条件1を返す。
    条件1がFalsyな場合は条件2を返す。
    よって、条件のどちらかがTruthyであればTruthyな値がリターンされ、両方ともがFalsyなときにはじめてFalsyな値がリターンされる。

If文 - If Statements

if文についてメモ。()の中がTruthyかFalsyかを比較してフローを制御する。

()の中がTruthyなら// do somethingへ進み、Falsyならelse以下へ進む。どれかに進んだあとはif文のフローから抜ける。


else ifの書き方。if()がFalsyなときにelse if()に進み、else if()の中がTruthyなときに// do hogeが実行される。


変数が存在しているかどうかのチェックの仕方についてメモ。

if(変数名)という書き方だと、その変数をスコープ内で宣言していなかったときにエラーがでてしまう。
存在するかのチェックをしたい場合はif(typeof 変数名 !== 'undefined')とすることで、エラーを回避できる。

ちなみに同じundefinedでも、変数を宣言しているけどundefinedな場合と、そもそも宣言すらしていなくてundefinedな場合では扱いが異なる。

変数の宣言はしてあるけどundefinedなときはif(変数名)でもエラーはでない。

そもそも変数を宣言すらしていない場合にif(変数名)でエラーになる。

  • sortメソッドは、ABC順で並び替える。その際に要素の値を自動的にStringに変換して比較する。

  • sortメソッドは対象の配列自体を並び替える。

  • sortメソッドで数値として比較して並び替えたい場合は、Compare functionを使う。


オブジェクトリテラル - Object Literals

オブジェクトとは、変数やメソッド、配列などの集まり。オブジェクトがもつ変数などの情報をプロパティと呼び、プロパティが関数の場合はメソッドと呼ぶ。オブジェクトの中に、プロパティの値としてさらに別のオブジェクトをもつこともできる。

以下オブジェクトの書き方についてのメモ。

  • プロパティにアクセスするにはオブジェクトの名前に.をつけて記述する。

  • []でもプロパティにアクセスできる。

  • メソッド内で現在のオブジェクトのプロパティにアクセスするにはthisを使う。（ただし、アロー関数ではthisの挙動が異なるので注意）


日付と時刻 - Dates & Times

JSには日付や時刻を扱うことが可能なDateオブジェクトが用意されている。

以下Dateオブジェクトの使い方についてメモ。

  • インスタンス生成時に日時を指定する場合、ブラウザごとに動作がことなるため注意が必要。

  • タイムゾーンに関しては、意図的に指定しない限りJSが動いているクライアントのタイムゾーンが使われる様子。

  • getMonth()では1月が0で、getDay()では日曜日が0であることに注意。
