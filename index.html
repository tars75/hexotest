イベントリスナー / Event Listeners

UIとWebページのインタラクションを実装するには、イベントリスナーを使う。

DOM上のエレメントなどにイベントリスナーをつけることで、そこで特定のイベントが発生したかどうかを拾って、イベントが発生したときの動作を決めることができる。

DOM上のすべてのエレメントに対してイベントリスナーをつけることができる。


書き方は下記の通り。

.addEventListener(パラメータ1, パラメータ2);

  • パラメータ1には何のイベントをトリガーにするかを指定。

  • パラメータ2にはコールバックする関数を指定。

  • イベントが発生した際は、まず関数がコールバックされ、そのあとにデフォルトのビヘイビアがあればそれが行われる。（例えばそのエレメントがリンクであれば、関数が実行された後にデフォルトのビヘイビアとしてリンク先のページに飛ぶ。）

  • 関数のパラメータにイベントオブジェクトを引数として指定でき、それに対して.preventDefault()を使うことでデフォルトのビヘイビアを防ぐことができる。

DOM - Document Object Model

DOMとは、ざっくり言うとHTMLドキュメントの構造モデル。
JSなどのプログラムからHTMLを取り扱えるようにするためのもの。
（HTMLだけでなくXMLとかもあるけど、基本ここではHTMLのDOMとして取り扱う。）

DOMでは、すべてのHTMLのエレメント（要素）やアトリビュート（属性）、CSSのスタイルなど、とにかくHTMLドキュメントの全てが、ノードの繋がりによるツリー構造で再現される。

Webページが読み込まれると、ブラウザはそのページのDOMを生成する。

JavaScriptについてのメモ第8回目。今回はいよいよWindowオブジェクトについて。

Window Object

Windowオブジェクトはブラウザのウインドウ本体を表し、ブラウザとJSをつなぐ一番の大本となるオブジェクト。

ブラウザやウェブページはすべて、オブジェクトのツリー構造によって表現されている。
「ウインドウのなかの、htmlエレメントのなかの、headエレメントの中の、pエレメントの〜、」といった形でブラウザやWebページの要素をオブジェクトのつらなりとして扱えるようになっていて、その一番上にあるのがこのWindowオブジェクト。

ウェブページのツリー構造の標準化されたモデル自体をDOMとよび、documentオブジェクトによってそのモデル構造が実現されている。
その中にはすべてのhtmlエレメントに対応したプロパティ、すべてのhtmlエレメントにアクセスできるメソッド、すべてのhtmlイベントなどが含まれている。
このdocumentオブジェクトもまた、Windowオブジェクトに含まれている。

とりあえずコンソールでwindowと入力してみると、ずらっとWindowオブジェクトに含まれているものの一覧がでてくる。

  • alert()
  • documentオブジェクト
  • fetch API
  • innerHeight
  • innerWidth

などなど。とにかく数え切れないくらいでてくる。

Windowオブジェクトのすべてをメモするのは不可能なので、今回はさらっと触りだけメモする。

関数宣言 - Function Declarations

関数とは、なんらかの機能をもったコードのかたまりのこと。基本的に関数を宣言するだけではそのコードは実行されない。関数を使いたいところで呼びだすことでそのコードを実行できる。

パラメーターを設定し、関数を呼びだすときに値（引数と呼ばれる）を渡して、その値を関数の中で使うことができる。また関数の中で処理した結果を出力して渡すこともできる。（返り値やreturnなどと呼ばれる。）

パラメーターは,で区切ることで複数設定でき、またパラメーターなしでも関数を作れる。

パラメーターのある関数を呼ぶときに、引数を何も渡さないと、パラメータはundefinedになる。

Truthy or Falsy

JSではすべての変数がTruthyかFalsyに分けられる。booleanに変換したときにTrueになる値がTruthy、Falseになる値がFalsyと呼ばれる。

Falsyな値：

  • false
  • 0 and -0
  • '' and "" (空っぽの文字列)
  • null and undefined
  • NaN
  • document.all

上記以外はすべてTruthyな値。
