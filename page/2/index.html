Numberへの変換

方法1 - Number()で囲む。

  • Numberに変換するとtrueは1になり、falseとnullは0になる。

  • 数値化できない場合のvalueは、NaN (Not a number)となる。Not a numberとは言っても第一回の記事でメモしたように、NaNのデータ型はNumber型であることに注意。

  • 1文字でも数字（および小数点、およびE+n表記）以外の文字列がある場合はNaNになる。ただし、+および-に関しては、それらが1番最初の文字の場合に限りNaNにならず数値化される。
    例）Number('+123.4')は123.4となるがNumber('123+4')はNaNとなる。


方法2 - parseInt()またはparseFloat()を使う。

  • parseInt()は整数に、parseFloat()は小数に変換する。

  • parseInt()に関しては本来は第2引数をもつが、今回はそこについては触れない。（基数という何進法として扱うかを設定できるパラメーターで、デフォルトでは10だが第1引数が0xで始まるときは16になる）

  • 最初の文字が数値に変換できない場合はNaNを返す。

  • Number()とは異なり、数値以外の文字列があったからといって必ずしもNaNにはならず、最初の文字から数値以外の文字列のところまではちゃんと数値化される。

  • Number()とは異なり、配列だからといって必ずしもNaNにはならず、最初の要素が数値化される。

JavaScriptについて学ぶ - 1

投稿日 2018-04-04 | In JavaScript , 基礎

JavaScriptについて新たに学習したことや、改めて復習したことなどを自分のためにメモしていく。第1回目。

JavaScriptを動かす

簡単にJavaScriptを動かす方法3つをメモ。

1. scriptエレメントの中に書いて、html文書に直接スクリプトを埋め込む。

HTML5ではscriptエレメントの中に書いたものはデフォルトでJSとして実行される。JSでスクリプトを書く場合、type属性をわざわざ書かずに省略することが推奨されている。

2. scriptエレメントのsrc属性を使って、外部から読み込む。

外部スクリプトを読み込んで実行する。htmlファイルとは別のファイルを作って、そこにjsを書く。この方法を使う場合は、1番目のやり方でscriptエレメントの中に直接スクリプトを書いてはいけない。

3. ブラウザのデベロッパーツールでコンソールを出して打ち込む。
コンソールにJSを入力することはあんまりないかもしれないけど、入力すればちゃんと動く。

変数 - var, let, const

変数には、var, let, constの3つがある。
ES6(ES2015)からブロックスコープの変数であるletとconstが追加された。
主要なブラウザでletとconstが使えるようになった現在では、グローバルスコープの変数であるvarの使用は非推奨となっている。

letとconstの比較。

  • let： 宣言できるのは一回のみ。reassign可（reassign = 代入しなおすこと）。
  • const： 宣言できるのは一回のみ。reassign不可。また、宣言時に初期値を代入しないとエラーになる。

ただし、constはreassign不可であるものの、配列やオブジェクトの場合はmutableである（値が可変であり、宣言したあとも中の値を変更できる可能性がある）ことに注意！！

変数名に使えるのは、次の三つ。

  • 英数字（英字に関しては、大文字と小文字は区別される）
  • _（アンダースコア）
  • $ （マネーサイン）

ただし、数字を一文字目にはできない。
また、$や_を一文字目にしてもエラーなどはでないものの、他のライブラリやフレームワーク独自の規格とかぶったりするので基本的には避けたほうがよいとされている。

変数名に複数の単語をもつ場合、従わなくてもエラーがでるわけではないけど書き方の作法のようなものがある。

  • Camel case
    基本的にはこの書き方を使う。
    例）firstName

  • Pascal case
    クラス名と、クラスのコンストラクタにはこの書き方を使う。
    例）FirstName

データ型

JavaScriptには、大きく分けてPrimitiveとObjectの二つのデータ構造がある。さらにPrimitiveであるデータ型は6つある。その6つ以外のものは全てObject型となる。

Primitives

オブジェクトではないデータのこと。基本的には値そのものを指す。メソッドは持たず、immutableな（変更することができない）値と定義されている。

  • String型
    文字列。ラッパーオブジェクトあり。

  • Number型
    数値。整数に限らず、小数や+/-Infinity、およびNaNも全てこのnumber型。ラッパーオブジェクトあり。

  • Boolean型
    trueまたはfalseのどちらかの値をもつ。ラッパーオブジェクトあり。

  • Null型
    意図的に変数の値が空であることを示す時に使う。値はnullのみ。注意点として、typeof nullのリターンがObjectになるというバグがある。

  • Undefined型
    何も代入されていない変数の値。

  • Symbols型(ES6)
    ES6からの新機能。Objectのプロパティを指すユニークなシンボル。ラッパーオブジェクトあり。

Objects

オブジェクトになるものの例。

  • Arrays（配列）
  • Object Literals（オブジェクトリテラル）
  • Functions（関数）
  • Dates（日付オブジェクト）
    …など。Primitiveでないものは全てObjectになる。


ラッパーオブジェクト

NullとUndefined以外はラッパーオブジェクトが用意されているため、オブジェクトのように扱うことができる。これによってもともと用意されている便利なビルトインメソッドやプロパティを使うことができる。


Immutableについて

immutabelという単語を目にしたときに「どいうこと？変数に再代入できないの？」という疑問が浮かんだが、変数に再代入することはできる。

試しにstrという変数に、'foo'という文字列を格納したあと'bar'を再代入してみる。

このとき何が行われているかというと、

  1. 'foo'を格納するためのメモリが割り当てられる。
  2. 'bar'を格納するために、新たに別のメモリが割り当てられる。
  3. 'foo'が格納されていたメモリはどうなるのかというと、メモリライフサイクルやGarbage Collectionと呼ばれる機能によって必要ないと判断されたときに自動的に解放される。

では、今度はメモリに格納されている値自体を変えようとしてみる。

最初にfを格納したメモリの値を変えようとしても、変えられないということがわかる。

つまり、Primitive値がimmutableというのは、変数のことではなくメモリに格納されている値そのものがimmutableであるということを言っている。


Dynamically Typed Language （動的型付け言語）

ここまでデータ型について記載したが、JSは動的型付けと呼ばれる言語のため自分でデータ型を指定する必要がない。以下それに関するメモ。

  • データ型は宣言によって決まるのではなく、変数の中の値によって自動的に決まる。

  • 同じ変数に別のデータ型が入る。例えば最初に数値を入れていた変数に対して、後から文字列を代入することができる。（自動的にNumberからStringに変換される。）

  • Primitive型でも必要に応じてラッパーオブジェクト（オブジェクト型）で自動的にラップされる。

  • Static Typing（静的型付け）をできるようにするスーパーセット（上位拡張）やアドオンがある。TypeScriptやFlowなど。また特に日本国内ではそういったものをJSのAlternativesとして、AltJSと呼んだりする。英語圏で一般的に使われているのかどうかは不明。（おそらく、少なくとも意味は通じると思う。。。）

curlについて調べる

投稿日 2018-04-02 | In コマンドライン

Webについて勉強しはじめたばかりだけどコマンドラインが関わってくると、かなりの頻度でこのcurlというコマンドがでてくる。curlってなんだ？？ってことで、ちょっと調べてみることにした。

なるべくあとで見返したときにわかりやすように、調べたことを自分の言葉でメモしていこうと思う。

curlとは？

curlとは、指定したURLに対していろんなタイプのリクエスト（GETとかPOSTとか）を行ったり、データをやりとり（ダウンロードとかアップロードとか）したりするのに使うツールのこと。

HTTPやらFTPなど、いろんなインターネットプロトコルに対応している。

全く同じ名前のCurl言語というプログラミング言語があるみたいだけど、それとは全く別物なので注意。

読み方はそのままカールでおk。またはシーユーアールエルと呼ばれることもある。

ここに公式のドキュメントがある。

curlの歴史

1996年にDaniel Stenbergさんという方が、IRC（チャットのアプリみたいなもの）で、為替レートを変換してくれるチャットボット（100ドルを円に変換して！といったら変換してくれるみたいなもの）を暇つぶしで作ろうとしたのがきっかけだそうな。

最初はHttpGet 1.0という名前で文字通りhttp-onlyのツールだったのが、その後にFTPダウンロードもサポートしたurlget 2.0になり、最終的に1998年にcurl 4という名前でリリースされたらしい。ちょうど20年前くらい。オープンソースで開発され、世界中のたくさんの有志の人たちが関わって作ったとのこと。

どういう風に使うの？

1. 準備

とりあえずここでは、コマンドラインで使う前提で進める。

最近のバージョンのOS X / macOSや、ほとんどのLinuxのディストリビューションであれば最初からインストールされていて、コマンドラインですぐにこのcurlというツールを使えるようになっている。Windowsの場合でもGit bashなどのコマンドラインツールであれば最初からcurlコマンドが使える。

とうことで、基本的にはWindows以外であれば特に準備はいらない。Windowsの場合でもcurlが使えるコマンドラインツールを使ったり、curl自体をダウンロードすれば普通に使えるようだ。

Themeを変える

投稿日 2018-04-02 | In Hexo

デフォルトのテーマはlandscapeというテーマだけど、もっと違うデザインを試してみたくなったので手っ取り早く他のテーマを入れてみる。

（ちなみに英語圏の人が喋ってるときの「theme」の発音はどう聴いても「シーム」とか、「スィーム」とかそんな感じだけど、日本でひろまってる「テーマ」という発音はドイツ語がもとらしい。）

Themeの変え方

  1. 公式のTheme一覧から新しいテーマを探す。

  2. Hexoブログのルートにある、themesというフォルダの中に、新しいテーマのフォルダを入れる。手動でダウンロードして入れてもいいし、git cloneしてもおk！

  3. Hexoブログのルートにある、_config.ymlファイルのなかの「theme: landscape」ってなってるところを、新しく入れたテーマの名前に書き換える。

ざっくり書くとこんな感じでThemeを変更できる。

  • ただし、Themeごとに独自で設定が必要だったり、あらたに専用のディレクトリやページを作る必要があったりするので、実際には上で書いたやり方だけではうまくいかない場合がほとんどの様子。

  • ThemeごとにREADMEが用意されていて、Themeによってはいろんな設定や機能があったりするのでREADMEは必ず見たほうがよさそう。

注意点

  • いいなと思ったThemeがあっても、READMEが中国語しかなかったりするのが結構つまづきポイントかも。

感想

Hexoに関して、中国語の情報が圧倒的に多くて、英語の情報があまりないように感じる。英語の公式ドキュメントもちょっと情報量が薄いような？とりあえず中国のコミュニティがとても盛り上がっている様子。

英語のコミュニティがもっと盛り上がってるもののほうが情報は探しやすそうなので、他の静的サイトジェネレータもちょっと調べてみようと思う。
