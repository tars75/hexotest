JavaScriptについて学ぶ - 1

投稿日 2018-04-04 | In JavaScript , 基礎

JavaScriptについて新たに学習したことや、改めて復習したことなどを自分のためにメモしていく。第1回目。

JavaScriptを動かす

簡単にJavaScriptを動かす方法3つをメモ。

1. scriptエレメントの中に書いて、html文書に直接スクリプトを埋め込む。

HTML5ではscriptエレメントの中に書いたものはデフォルトでJSとして実行される。JSでスクリプトを書く場合、type属性をわざわざ書かずに省略することが推奨されている。

2. scriptエレメントのsrc属性を使って、外部から読み込む。

外部スクリプトを読み込んで実行する。htmlファイルとは別のファイルを作って、そこにjsを書く。この方法を使う場合は、1番目のやり方でscriptエレメントの中に直接スクリプトを書いてはいけない。

3. ブラウザのデベロッパーツールでコンソールを出して打ち込む。
コンソールにJSを入力することはあんまりないかもしれないけど、入力すればちゃんと動く。

変数 - var, let, const

変数には、var, let, constの3つがある。
ES6(ES2015)からブロックスコープの変数であるletとconstが追加された。
主要なブラウザでletとconstが使えるようになった現在では、グローバルスコープの変数であるvarの使用は非推奨となっている。

curlについて調べる

投稿日 2018-04-02 | In コマンドライン

Webについて勉強しはじめたばかりだけどコマンドラインが関わってくると、かなりの頻度でこのcurlというコマンドがでてくる。curlってなんだ？？ってことで、ちょっと調べてみることにした。

なるべくあとで見返したときにわかりやすように、調べたことを自分の言葉でメモしていこうと思う。

curlとは？

curlとは、指定したURLに対していろんなタイプのリクエスト（GETとかPOSTとか）を行ったり、データをやりとり（ダウンロードとかアップロードとか）したりするのに使うツールのこと。

HTTPやらFTPなど、いろんなインターネットプロトコルに対応している。

全く同じ名前のCurl言語というプログラミング言語があるみたいだけど、それとは全く別物なので注意。

読み方はそのままカールでおk。またはシーユーアールエルと呼ばれることもある。

ここに公式のドキュメントがある。

curlの歴史

1996年にDaniel Stenbergさんという方が、IRC（チャットのアプリみたいなもの）で、為替レートを変換してくれるチャットボット（100ドルを円に変換して！といったら変換してくれるみたいなもの）を暇つぶしで作ろうとしたのがきっかけだそうな。

最初はHttpGet 1.0という名前で文字通りhttp-onlyのツールだったのが、その後にFTPダウンロードもサポートしたurlget 2.0になり、最終的に1998年にcurl 4という名前でリリースされたらしい。ちょうど20年前くらい。オープンソースで開発され、世界中のたくさんの有志の人たちが関わって作ったとのこと。

どういう風に使うの？

1. 準備

とりあえずここでは、コマンドラインで使う前提で進める。

最近のバージョンのOS X / macOSや、ほとんどのLinuxのディストリビューションであれば最初からインストールされていて、コマンドラインですぐにこのcurlというツールを使えるようになっている。Windowsの場合でもGit bashなどのコマンドラインツールであれば最初からcurlコマンドが使える。

とうことで、基本的にはWindows以外であれば特に準備はいらない。Windowsの場合でもcurlが使えるコマンドラインツールを使ったり、curl自体をダウンロードすれば普通に使えるようだ。

Themeを変える

投稿日 2018-04-02 | In Hexo

デフォルトのテーマはlandscapeというテーマだけど、もっと違うデザインを試してみたくなったので手っ取り早く他のテーマを入れてみる。

（ちなみに英語圏の人が喋ってるときの「theme」の発音はどう聴いても「シーム」とか、「スィーム」とかそんな感じだけど、日本でひろまってる「テーマ」という発音はドイツ語がもとらしい。）

Themeの変え方

  1. 公式のTheme一覧から新しいテーマを探す。

  2. Hexoブログのルートにある、themesというフォルダの中に、新しいテーマのフォルダを入れる。手動でダウンロードして入れてもいいし、git cloneしてもおk！

  3. Hexoブログのルートにある、_config.ymlファイルのなかの「theme: landscape」ってなってるところを、新しく入れたテーマの名前に書き換える。

ざっくり書くとこんな感じでThemeを変更できる。

  • ただし、Themeごとに独自で設定が必要だったり、あらたに専用のディレクトリやページを作る必要があったりするので、実際には上で書いたやり方だけではうまくいかない場合がほとんどの様子。

  • ThemeごとにREADMEが用意されていて、Themeによってはいろんな設定や機能があったりするのでREADMEは必ず見たほうがよさそう。

注意点

  • いいなと思ったThemeがあっても、READMEが中国語しかなかったりするのが結構つまづきポイントかも。

感想

Hexoに関して、中国語の情報が圧倒的に多くて、英語の情報があまりないように感じる。英語の公式ドキュメントもちょっと情報量が薄いような？とりあえず中国のコミュニティがとても盛り上がっている様子。

英語のコミュニティがもっと盛り上がってるもののほうが情報は探しやすそうなので、他の静的サイトジェネレータもちょっと調べてみようと思う。

Markdownのテスト

投稿日 2018-03-31 | In Markdown

Markdownはまったく書いたことないので、とりあえずいろいろ試して見る。いろいろ方言があるみたいだけど、今回はHexoでどうなるのか確認もしたい。ちなみに困ったときはそのままHTMLのタグを書けばちゃんと反映されるみたい。

Headingの書き方

Headingはでっかい見出し文字のこと。#が多いほど小さくなる。とてもよく使いそう。

Italic と Strong

Italicは斜体で、Strongは強調（太字）のこと。あまり使わないかな？

Strikethrough

Strikethroughとは打ち消し線のこと。これもあんまり使わないかも。

Horizontal Rule

Horizontal Ruleとは水平のけい線のこと。コンテンツを分けるときに使う。

Blockquote

Blockquoteとは、引用文のこと。わりとよく見かける気がする。

Link

Linkとは、、文字通りリンクそのまま。

UL

ULとは番号順なしのリストのこと。

OL

OLとは番号順ありのリストのこと。ULの * を 1. に変えるだけ。

Images

Imagesはそのまんま画像のこと。リンクの書き方にプラスして、先頭に!をつけると画像になる。
